# Command: volume claim list

The `volume claim list` command displays all task group host volume claims. These claims
are used by [stateful deployments].

## Usage

```plaintext
nomad volume claim list [options]
```

This command accepts an optional claim ID or prefix as the sole argument. If
there is an exact match based on the provided volume ID or prefix, then
information about the specific claim is queried and displayed. Otherwise, a
list of matching claims and information will be displayed.

If the ID is omitted, the command lists out all of the existing claims and a
few of the most useful status fields for each.

When ACLs are enabled, this command requires a token with the `host-volume-read`
capability in the volume's namespace.

## General Options

@include 'general_options.mdx'

## List Options
  
- `-job`: Show only volume claims from a particular job ID.

- `-group`: Show only volume claims from a particular task group name.

- `-volume-name`: Show only volume claims from a particular volume name.

- `-verbose`: Display full information.

## Example

List of all task group host volume claim for job ID "example".

```shell-session
$ nomad volume claim list -job=example
ID        Namespace  Job ID   Volume ID  Volume Name
81b43af2  default    example  18377894   sticky-volume
7ce38017  default    example  c3c27514   example-volume
```
